INSTRUCTION 2.1 — Implement src/retraining/drl_retrainer.py
-------------------------------------------------------------
Class: DRLRetainer (inherits from pydantic BaseModel for config validation)

  Fields (validated via pydantic):
    - model_type: Literal["PPO", "SAC", "TD3", "DDPG"] = "PPO"
    - total_timesteps: int = Field(ge=100_000, default=500_000)
    - eval_episodes: int = Field(ge=10, default=100)
    - reward_threshold: float = Field(gt=0.0, default=1.5)
    - max_retrain_iterations: int = Field(ge=1, le=10, default=3)
    - data_window_days: int = Field(ge=30, default=90)
    - colab_gpu: Literal["T4", "V100", "A100"] = "T4"
    - checkpoint_dir: Path
    - retrain_trigger: Literal["scheduled", "drift_detected", "manual"]

  Methods:
    - load_latest_model(self) -> stable_baselines3.common.base_class.BaseAlgorithm
      * Load from checkpoint_dir, raise FileNotFoundError if absent
      * Validate model zip integrity before loading
      * Wrap in try/except with structlog error logging

    - detect_model_drift(self, recent_rewards: list[float]) -> bool
      * Implement KL-divergence drift detection per [B2] Ch.9
      * Compare recent episode reward distribution vs. baseline
      * Trigger retrain if KL > threshold (configurable, default 0.15)
      * Log drift metrics to structlog

    - prepare_training_data(self, end_date: date) -> gymnasium.Env
      * Fetch last N days of market data via httpx (NSE/BSE APIs)
      * Apply feature engineering pipeline (returns, volatility, OI, Greeks)
      * Wrap in custom gymnasium.Env with Indian market constraints:
        - Lot sizes per NSE specifications
        - Margin requirements per SEBI/Span margin
        - Trading hours (9:15-15:30 IST)
        - Circuit breaker rules
      * Validate env observation/action spaces

    - retrain(self, resume_from_checkpoint: bool = True) -> dict
      * Main retrain orchestration
      * If resume_from_checkpoint: load existing model, continue training
      * Else: initialize fresh model
      * Train with EvalCallback (eval_freq=5000, n_eval_episodes=eval_episodes)
      * Save checkpoints every 50k steps
      * After training, run eval_episodes; if mean_reward < reward_threshold,
        re-train up to max_retrain_iterations
      * Return metrics dict: {"mean_reward", "std_reward", "timesteps",
        "drift_before", "drift_after", "retrain_iterations", "model_version"}
      * All exceptions caught, logged via structlog, re-raised as
        DRLRetrainError (custom exception)

    - export_for_colab(self, output_path: Path) -> None
      * Serialize model + config + normalization stats into .zip
      * Generate Google Colab notebook (scripts/colab_retrain_notebook.ipynb)
        with cells for:
          1. T4 GPU check (torch.cuda.is_available())
          2. Dependency install (!pip install stable-baselines3[extra])
          3. Data download from cloud storage
          4. Model retrain loop with EvalCallback
          5. TensorBoard logging
          6. Export updated model back to cloud
      * Validate .zip integrity post-creation

    - validate_retrained_model(self, old_model_path: Path, new_model_path: Path)
        -> bool
      * Backtest both models on same OOS window (last 30 days)
      * Per [B5] Ch.12: Use combinatorial purged cross-validation
      * New model must pass all checks:
        1. Sharpe ratio >= old model's Sharpe - 0.1 (no regression)
        2. Max drawdown <= old model's max DD + 0.02
        3. Win rate >= old model's win rate - 0.05
        4. No NaN/Inf in predictions
      * If any check fails: log warning, keep old model, flag for review
      * Return True only if all checks pass

INSTRUCTION 3.1 — Implement src/retraining/sentiment_updater.py
----------------------------------------------------------------
Class: SentimentUpdater (pydantic BaseModel)

  Fields:
    - model_name: str = "finiteautomata/bertweet-base-sentiment-analy"
    - indian_market_model: str = "custom/indian-market-sentiment-v2"
    - data_sources: list[Literal["moneycontrol", "economictimes",
        "livemint", "ndtv_profit", "reuters_india"]]
    - min_new_articles: int = Field(ge=100, default=500)
    - fine_tune_epochs: int = Field(ge=1, le=10, default=3)
    - learning_rate: float = Field(gt=0.0, default=2e-5)
    - batch_size: int = Field(ge=8, default=16)
    - validation_split: float = Field(gt=0.0, lt=1.0, default=0.2)
    - checkpoint_dir: Path
    - label_schema: list[str] = ["bullish", "bearish", "neutral"]

  Methods:
    - fetch_new_articles(self, since: datetime) -> datasets.Dataset
      * Scrape/configure API calls to each data_source via httpx
      * Apply rate limiting with tenacity (retry + backoff)
      * Filter for Indian market keywords: NIFTY, SENSEX, RBI, SEBI,
        F&O, quarterly results, corporate actions, IPO, etc.
      * Deduplicate by URL + title similarity (cosine > 0.95)
      * Return HuggingFace Dataset with columns: text, source, date, url

    - label_articles(self, dataset: Dataset) -> Dataset
      * Use existing model for pseudo-labeling on high-confidence samples
      * Flag low-confidence (< 0.7) for manual review
      * Apply Indian market sentiment heuristics:
        - RBI rate hike → bearish bias
        - FII net buyer → bullish bias
        - DII net seller → mixed signal
        - Earnings beat → bullish for sector
      * Validate label distribution (no class > 70% without flagging)

    - fine_tune(self, labelled_data: Dataset) -> dict
      * Load base model + tokenizer
      * Apply LoRA/PEFT for parameter-efficient fine-tuning
      * Split into train/val per validation_split
      * Train with HuggingFace Trainer, early stopping (patience=2)
      * Evaluate on held-out validation set
      * Metrics: accuracy, F1 (macro), precision, recall per class
      * Save model + tokenizer + label encoder to checkpoint_dir
      * Return metrics dict

    - validate_updated_model(self) -> bool
      * Compare new model vs. old on curated Indian market test set
      * Test set: 500 manually verified articles (curated quarterly)
      * Must meet:
        1. Macro F1 >= old model F1 - 0.02
        2. No class precision < 0.7
        3. Calibration error (ECE) <= 0.1
      * Log confusion matrix to structlog
      * Return True only if all thresholds met

    - update_production_model(self) -> None
      * Atomic swap: write new model to temp dir, rename to production path
      * Invalidate any caching layers
      * Log model version transition
      * If swap fails: rollback to previous model immediately

INSTRUCTION 4.1 — Implement src/retraining/rag_knowledge_refresh.py
--------------------------------------------------------------------
Class: RAGKnowledgeRefresher (pydantic BaseModel)

  Fields:
    - vector_store_path: Path
    - embedding_model: str = "sentence-transformers/all-MiniLM-L6-v2"
    - chunk_size: int = Field(ge=100, le=2000, default=512)
    - chunk_overlap: int = Field(ge=0, default=50)
    - knowledge_sources: list[str] = [
        "SEBI circulars", "RBI monetary policy", "NSE bulletins",
        "BSE notices", "corporate actions", "economic indicators",
        "futures & options data", "sector analysis reports"]
    - max_document_age_days: int = Field(ge=30, default=365)
    - similarity_threshold: float = Field(gt=0.0, lt=1.0, default=0.85)
    - collection_name: str = "indian_market_knowledge"

  Methods:
    - scan_stale_documents(self) -> list[dict]
      * Query ChromaDB for documents older than max_document_age_days
      * Check source URLs for updates (ETag/Last-Modified via httpx HEAD)
      * Return list of stale/outdated documents with metadata

    - fetch_updated_documents(self, stale_docs: list[dict]) -> list[Document]
      * Re-download updated versions from sources
      * Extract text from PDF, HTML, plain text
      * Apply Indian market context tagging (NSE/BSE/SEBI category)
      * Validate document integrity (non-empty, valid encoding, no corruption)

    - deduplicate_and_chunk(self, documents: list[Document]) -> list[Document]
      * Embed all documents, find near-duplicates (cosine > similarity_threshold)
      * Remove duplicates, keep most recent
      * Split into chunks with RecursiveCharacterTextSplitter
        (chunk_size, chunk_overlap)
      * Attach metadata: source, date, category, version

    - update_vector_store(self, chunks: list[Document]) -> dict
      * Delete stale document chunks from ChromaDB
      * Insert new/updated chunks
      * Persist vector store to disk
      * Validate: query test questions, verify retrieval quality
        (must return relevant docs for 10 pre-defined test queries)
      * Return stats: {"added", "deleted", "updated", "total_chunks",
        "validation_pass"}

    - generate_knowledge_report(self) -> dict
      * Count documents by source, category, age distribution
      * Identify gaps (sources with no recent updates > 60 days)
      * Compute embedding coverage statistics
      * Return structured report dict

  Per [B7] Ch.9 — Iron Condor = Bull Put Spread + Bear Call Spread

  Fields:
    - underlying: str (e.g., "NIFTY", "BANKNIFTY")
    - expiry: date
    - wing_width_pct: float = Field(gt=0, default=2.0)   # % from ATM
    - short_put_delta: float = Field(ge=-1.0, lt=0, default=-0.20)
    - short_call_delta: float = Field(gt=0, le=1.0, default=0.20)
    - long_put_delta: float = Field(ge=-1.0, lt=-0.20, default=-0.10)
    - long_call_delta: float = Field(gt=0.20, le=1.0, default=0.10)
    - lot_size: int = Field(gt=0)          # Populated from NSE spec
    - margin_pct: float = Field(gt=0, default=1.0)  # SPAN + exposure margin
    - max_profit_pct: float (computed)
    - max_loss_pct: float (computed)
    - breakeven_points: tuple[float, float] (computed)

  Methods:
    - validate_strike_selection(self, spot: float, chain: dict) -> bool
      * Verify strikes exist in NSE option chain
      * Verify strikes have sufficient liquidity (OI > 10000, volume > 100)
      * Verify wing widths are consistent
      * Verify net premium is positive (credit spread)
      * Raise IronCondorValidationError if any check fails

    - compute_greeks(self, spot: float, iv_surface: dict) -> Greeks
      * Compute net delta, gamma, theta, vega for the 4-leg position
      * Use Black-Scholes with Indian market adjustments:
        - Risk-free rate: RBI 91-day T-bill yield
        - Dividend yield: NIFTY50 dividend yield
        - Settlement: T+1 for options
      * Return Greeks dataclass

    - compute_margin(self) -> float
      * Calculate SPAN margin + exposure margin per NSE framework
      * Validate against broker API (if available) or NSE margin calculator
      * Apply margin buffer of 10% for safety
      * Return total margin required per lot

    - generate_order_legs(self) -> list[OrderLeg]
      * Return 4 OrderLeg objects:
        1. SELL PUT (short_put_strike)
        2. BUY PUT (long_put_strike, further OTM)
        3. SELL CALL (short_call_strike)
        4. BUY CALL (long_call_strike, further OTM)
      * Each OrderLeg: symbol, side, strike, option_type, quantity, order_type
      * Validate: net order = credit (premium received > 0)

    - risk_assessment(self, spot: float, scenarios: list[dict]) -> RiskReport
      * P&L at: current spot, ±1σ, ±2σ, ±3σ moves
      * Max profit zone (between short strikes)
      * Max loss zones (beyond long strikes)
      * Probability of profit (POP) using lognormal distribution
      * Return RiskReport dataclass

    - should_adjust(self, current_spot: float, days_to_expiry: int,
        net_delta: float) -> AdjustAction
      * Adjustment rules:
        - If days_to_expiry < 7 and net_delta > 0.15: roll tested side
        - If loss > 50% of max profit: close or roll
        - If IV rank drops below 25: consider closing early
      * Return AdjustAction enum: HOLD, ROLL, CLOSE, ADJUST

      * CRITICAL: highlight unlimited risk zone beyond short strike
      * Compute: max profit, break-even, worst-case loss
      * POP with explicit warning about tail risk
      * Flag if max_loss > max_risk_multiplier * max_profit

Class: MarginCalculator

  Methods:
    - calculate_span_margin(self, positions: list[Position]) -> float
      * Simplified SPAN margin calculation per NSE framework:
        - Scan risk = max(loss across scenarios)
        - Scenarios: price up/down by 3σ, vol up/down by 3σ
        - Short option minimum charge
        - Inter-commodity spread credit
      * Return total SPAN margin

    - calculate_exposure_margin(self, positions: list[Position]) -> float
      * Exposure margin = % of MTM loss (typically 3% for options)
      * Apply per NSE specification for each position type

    - calculate_total_margin(self, positions: list[Position]) -> MarginBreakdown
      * Total = SPAN + Exposure + Premium (for buy positions)
      * Apply margin benefits for spread positions:
        - Iron condor: only max risk of either wing
        - Calendar spread: reduced margin per NSE offset benefit
        - Ratio spread: SPAN + exposure with short option charge
      * Return MarginBreakdown: span, exposure, premium, total, benefit

    - validate_margin_available(self, required: float, available: float)
        -> bool
      * Check required <= available * 0.80 (keep 20% buffer)
      * Raise InsufficientMarginError if not

INSTRUCTION 10.1 — Create Makefile with All Verification Targets
-----------------------------------------------------------------
  .PHONY: all lint typecheck security test coverage benchmark bom audit clean

  all: lint typecheck security test coverage bom

  lint:
      ruff check src/ tests/ --config pyproject.toml
      ruff format --check src/ tests/ --config pyproject.toml

  typecheck:
      mypy src/ --config-file pyproject.toml --strict

  security:
      bandit -r src/ -c pyproject.toml -f json -o reports/bandit.json
      pip-audit --desc --output reports/pip-audit.json
      gitleaks detect --source . --report-format json --report-path reports/gitleaks.json
      trivy fs --format json --output reports/trivy.json .

  test:
      pytest tests/ -v --tb=short -n auto

  coverage:
      pytest tests/ -v --cov=src --cov-report=term-missing \
        --cov-report=html:reports/coverage --cov-fail-under=85 -n auto

  benchmark:
      pytest tests/ -v --benchmark-only --benchmark-autosave

  bom:
      cyclonedx-bom -o SBOM/sbom.json --format json
      cyclonedx-bom -o SBOM/sbom.xml --format xml

  audit: security
      python -c "import json; data=json.load(open('reports/bandit.json')); \
        assert data.get('results',[])==[], 'Bandit findings detected'"
      python -c "import json; data=json.load(open('reports/pip-audit.json')); \
        assert len(data.get('dependencies',[]))==0, 'CVEs detected'"

  clean:
      ruff cache clean
      mypy --cache-clear
      Remove-Item -Recurse -Force reports/, .pytest_cache/, .mypy_cache/
