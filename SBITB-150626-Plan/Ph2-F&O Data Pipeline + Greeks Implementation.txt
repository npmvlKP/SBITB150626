-- View: ATM strikes per day (closest strike to spot close)
CREATE OR REPLACE VIEW v_atm_strikes AS
SELECT
    f.date,
    f.symbol,
    f.expiry,
    s.close AS spot_close,
    f.strike AS atm_strike,
    f.option_type,
    f.close AS option_close,
    f.volume,
    f.oi,
    ABS(f.strike - s.close) AS strike_distance
FROM fo_options_eod f
JOIN cm_spot_eod s ON f.date = s.date AND f.symbol = REPLACE(s.symbol, ' 50', '') AND f.symbol = REPLACE(s.symbol, ' BANK', '')
WHERE ABS(f.strike - s.close) = (
    SELECT MIN(ABS(f2.strike - s2.close))
    FROM fo_options_eod f2
    JOIN cm_spot_eod s2 ON f2.date = s2.date
    WHERE f2.date = f.date
      AND f2.symbol = f.symbol
      AND f2.expiry = f.expiry
      AND f2.option_type = f.option_type
      AND s2.symbol = s.symbol
);

Ph#2-4—src/data/event_log.py — Immutable Event Log (Kleppmann Ch.3)
This is the single source of truth for all market data. All pipelines write events here first; derived tables are populated from this log.

Requirements
Dataclass MarketEvent:
event_id: UUID — auto-generated uuid4
event_time: datetime — timezone-aware (IST/UTC)
event_type: str — one of: FO_BHAVCOPY, CM_BHAVCOPY, WS_TICK, WS_OI_UPDATE, GREEKS_SNAPSHOT
schema_version: int — starts at 1; incremented on schema change
payload: dict — the actual data (JSONB)
source: str — jugaad_data, kite_ws, computed
ingest_id: UUID — unique per pipeline run (idempotency: same ingest_id + event_id = no-op)

Class EventLogWriter:
__init__(self, db_url: str, batch_size: int = 5000)
async def append(self, event: MarketEvent) -> None
INSERT into market_events with ON CONFLICT (event_id, event_time) DO NOTHING (idempotent)
Buffer events in memory; flush when buffer >= batch_size or on explicit flush()
async def flush(self) -> int — returns rows written
async def close(self) -> None — flush remaining + close connection
Uses psycopg with COPY-style batch inserts for performance
Structlog on every flush: {"action": "event_log_flush", "rows": N, "event_types": [...]}
Class EventLogReader:
async def query(self, event_type: str | None, start: datetime, end: datetime, source: str | None = None, limit: int = 10000) -> list[MarketEvent]

Class EventCodec:
encode(event: MarketEvent) -> dict — serialize with schema_version
decode(raw: dict) -> MarketEvent — deserialize; handle schema migrations
Version migration registry: dict[int, Callable] mapping old schema_version → migration function
Current schema version: CURRENT_SCHEMA_VERSION = 1
Schema Evolution Example (Kleppmann Ch.4):
If v2 adds oi_change to FO_BHAVCOPY payload: migrate_v1_to_v2(payload) -> dict fills oi_change=0 for v1 events
decode() checks schema_version; if < current, applies all migrations in sequence
All methods fully type-hinted, async, with structlog. Every append must be idempotent (same event_id = no-op).
