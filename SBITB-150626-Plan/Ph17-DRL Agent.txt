 SECTION 0 — CRITICAL CORRECTIONS
═══════════════════════════════════════════════════════════════════════════════

  CC-01  DRL is SECOND OPINION, not sole strategy
         Weight starts at 0% (DISABLED), ramps to 15% (ACTIVE), caps at 25%
         (ESCALATED_2). Never exceeds 25% under any condition.

  CC-02  No Stable-Baselines3, Ray RLlib, or FinRL
         These add opaque abstraction layers that conflict with Phase 2-9
         infrastructure. Use CleanRL-style single-file PPO (~300 LOC) for
         full transparency and auditability (Zai & Brown Ch.4 pattern).

  CC-03  Custom Gymnasium env only — IndianMarketEnv
         FinRL's env classes enforce rigid observation/action spaces that
         cannot represent our 23-feature, 3-action, masked paradigm.
         Build from gymnasium.Env base class.

  CC-04  ONNX export for production inference, NOT PyTorch
         PyTorch CPU inference ~200ms per step. ONNX Runtime CPU ~15-40ms.
         Training uses PyTorch on Colab T4; production uses ONNX on Win11.
         Target: <50ms per inference step on CPU.

  CC-05  Episode = one trading day; step = one 5-min bar
         Aligns with Phase 11 eval_cycle=5min. NSE session 09:15-15:30
         = 75 bars/day. Episode terminates on RiskManager kill_switch
         or daily_loss_limit breach (Phase 10 constraints).

  CC-06  Observation normalization from training stats only
         Never recompute normalization on live data. Store running mean/std
         from training. Apply fixed normalization at inference time.
         Violating this causes distribution shift → silent PPO failure.

  CC-07  Action masking, NOT penalty-based invalid action handling
         Zai & Brown Ch.3: masking is superior to penalty because penalties
         require exploration of invalid actions to learn they are bad.
         Masking simply prevents them. Use MaskedCategorical distribution.

  CC-08  NIFTY lot size = 25 (not 50)
         Corrected in Phase 5. All reward calculations, position sizing,
         and env step logic must use lot_size=25.

  CC-09  DSR (Decision-Sharpe Ratio) as primary metric
         Phase 6 defines DSR. DRL agent must report DSR alongside
         Sharpe, Sortino, max drawdown, and Calmar ratio. DSR accounts
         for number of independent bets — critical for intraday evaluation.

  CC-10  Purge + embargo on train/validate split
         Phase 6 defines purge (remove overlapping bars) and embargo
         (minimum gap between train and validate). Apply to episode-level
         splits: 2022-2023 train, 2024 train, 2024-2025 OOS validate.
         5-day embargo between splits.

  CC-11  No retraining on live data
         Training happens ONLY on Colab with historical data. Production
         agent is frozen. Retraining requires full pipeline: train on
         Colab → validate OOS → export ONNX → deploy. No online updates.

  CC-12  Gradient clipping at 0.5
         PPO with unclipped gradients diverges on financial data.
         Zai & Brown Ch.4: clip_norm=0.5 is standard for PPO.
         Also clip rewards to [-10, 10] to prevent outlier-driven collapse.

  CC-13  GAE lambda = 0.95, gamma = 0.99
         Standard PPO parameters. Gamma=0.99 because intraday episodes
         are short (75 steps). Lambda=0.95 balances bias-variance in
         advantage estimation (Sutton & Barto Ch.12).

  CC-14  No reward shaping beyond PnL + risk penalty
         Shaped rewards (e.g., "reward for holding winning position")
         create unexpected agent behaviors. Use:
           reward = realized_pnl_this_step - risk_penalty
         where risk_penalty = 0.001 * |position_value| per step.
         Simple, auditable, no hidden incentives.

  CC-15  Checkpoint every 100k steps to Google Drive on Colab
         Colab free tier has ~12hr max session, T4 GPU 16GB VRAM.
         Checkpoint: model weights, optimizer state, normalization stats,
         training metrics JSON. Resume from latest checkpoint.

  CC-16  SignalSource.DRL as 6th member of SignalSource enum
         Phase 11 defines: TECHNICAL, VOLUME, SENTIMENT, MARKET_STRENGTH, RAG
         DRL is added as 6th. Not a Strategy (Phase 3 StrategyInterface);
         it is a signal source consumed by OrchestrationEngine.

  CC-17  OrchestrationSettings weight rebalancing required
         Current 5 weights sum to 1.0. Adding DRL weight requires
         proportional reduction of all 5 existing weights:
           new_w_i = old_w_i * (1 - drl_weight) / sum(old_weights)
         DRL weight transitions via state machine (CC-01).

  CC-18  No GPU/CUDA dependency in production code
         Production runs on Windows 11 CPU only. ONNX Runtime uses CPU
         execution provider. Any import of torch in production code
         is a hard error (caught by import linting in CI).

  CC-19  Seed everything for reproducibility
         torch.manual_seed, np.random.seed, random.seed, gymnasium.seed
         All set from config. Log seed in training metadata. Required for
         audit trail and regression testing.

  CC-20  Episode boundary = NSE trading calendar
         Episodes must respect NSE trading calendar (exclude holidays,
         half-days). Use same calendar as Phase 6 backtesting.

  CC-21  Feature staleness detection
         If any observation feature is older than 5 minutes (one bar),
         emit warning and use last known value. If older than 15 minutes,
         force SELL/FLAT action via masking. Never feed stale data.

  CC-22  No leverage in DRL env
         DRL agent trades 1 lot per action. Position sizing is handled
         by RiskManager (Phase 10), not by the DRL agent. Agent decides
         direction only: LONG, SHORT, FLAT.

  CC-23  Maximum episode steps = 75 (09:15 to 15:30, 5-min bars)
         Hard cap. If RiskManager hasn't terminated episode, force
         FLAT at step 75 (15:25-15:30 bar = final exit window).
         No overnight positions from DRL agent.

  CC-24  Validation uses walk-forward, NOT random split
         Phase 6 walk-forward: train on expanding window, validate on
         next period. 2022→2023 train, validate on Q1-2024. Then
         2022-2023+Q1-2024 train, validate on Q2-2024. Etc.

  CC-25  Entropy coefficient schedule, not fixed
         Start entropy_coef=0.01, linearly decay to 0.001 over
         total training steps. Prevents premature convergence to
         deterministic policy (Zai & Brown Ch.4).

  CC-26  Target KL divergence = 0.015 for early stopping
         If approx_kl > 0.015, stop PPO update epoch early.
         Prevents destructive policy updates (Schulman et al. 2017).

  CC-27  No custom CUDA kernels or Triton ops
         Must run on Colab T4 with standard PyTorch CUDA. No custom
         compiled ops that break portability or Colab compatibility.

  1.1  Sutton & Barto — Reinforcement Learning: An Introduction (2018)
  ─────────────────────────────────────────────────────────────────────

  | Ch  | Concept              | Application in Phase 17                          |
  |-----|----------------------|--------------------------------------------------|
  | 3   | Policy Gradient      | PPO is a policy gradient method. Log-prob        |
  |     |                      | objective with clipped surrogate.                |
  | 4   | Temporal Difference  | GAE (lambda-returns) is TD(lambda). Advantage    |
  |     |                      | = sum of gamma-discounted TD residuals.          |
  | 5   | N-step returns       | GAE generalizes n-step returns with lambda       |
  |     |                      | interpolation. lambda=0.95 approx many-step.     |
  | 12  | Eligibility traces   | GAE is the forward-view equivalent of eligibility |
  |     |                      | traces. No backward-view needed in PPO.          |
  | 13  | Policy gradient      | REINFORCE → Actor-Critic → PPO progression.      |
  |     |                      | PPO clips ratio r_t(theta) in [1-eps, 1+eps].   |

  Key equations (for implementation reference):
  * Advantage: A_t = SUM_{l=0}^{T-t-1} (gamma*lambda)^l * delta_{t+l}
    where delta_t = r_t + gamma*V(s_{t+1}) - V(s_t)
  * PPO clip: L^CLIP = E_t[min(r_t(theta)*A_t, clip(r_t(theta), 1-eps, 1+eps)*A_t)]
  * Entropy bonus: L^S = -E_t[pi_theta(a_t|s_t) * log pi_theta(a_t|s_t)]
  * Total loss: L = L^CLIP - c_1 * L^VF + c_2 * L^S
    where c_1=0.5 (value loss coef), c_2=entropy_coef (scheduled)

  4.4  step(action):

       def step(self, action):
           # 1. Validate action against mask (CC-07)
           if not self._valid_mask()[action]:
               action = 0  # Force FLAT if invalid

           # 2. Execute action -> position transition
           prev_position = self.current_position
           reward, trade_info = self._execute_action(action)

           # 3. Check RiskManager kill conditions (Phase 10)
           if self.risk_manager.kill_switch_active:
               self.done = True
               reward += self._force_close_pnl()

           # 4. Check daily loss limit (Phase 10)
           if self.episode_pnl < -self.risk_manager.daily_loss_limit:
               self.done = True
               reward += self._force_close_pnl()

           # 5. Check max steps (CC-23)
           self.step_count += 1
           if self.step_count >= self.max_steps:
               self.done = True
               reward += self._force_close_pnl()

           # 6. Get next observation
           self.current_bar_idx += 1
           obs = self._get_observation()
           mask = self._get_mask()

           # 7. Clip reward (CC-12)
           reward = np.clip(reward, -10.0, 10.0)

           info = {
               "mask": mask,
               "pnl": self.episode_pnl,
               "position": self.current_position,
               "trade": trade_info,
               "step": self.step_count,
           }
           return obs, reward, self.done, False, info

  4.5  Action masking logic (CC-07):

       def _get_mask(self) -> np.ndarray:
           """Returns boolean mask [FLAT_valid, LONG_valid, SHORT_valid]"""
           mask = np.ones(3, dtype=bool)

           # Cannot go LONG if already LONG (no pyramid, CC-22)
           if self.current_position == 1:
               mask[1] = False  # Cannot go LONG again
               mask[2] = False  # Must FLAT first (no reversal)

           # Cannot go SHORT if already SHORT
           if self.current_position == -1:
               mask[2] = False  # Cannot go SHORT again
               mask[1] = False  # Must FLAT first (no reversal)

           # Staleness: if features stale > 15min, force FLAT only (CC-21)
           if self._feature_staleness() > self.config.feature_staleness_force_s:
               mask = np.array([True, False, False], dtype=bool)

           return mask

  4.6  Reward function (CC-14):

       def _execute_action(self, action) -> tuple[float, dict]:
           reward = 0.0
           trade_info = {}

           if action == 0:  # FLAT
               if self.current_position != 0:
                   # Close position
                   pnl = self._calculate_pnl()
                   reward = pnl
                   self.episode_pnl += pnl
                   self.current_position = 0
                   self.entry_price = 0.0
                   trade_info = {"action": "close", "pnl": pnl}

           elif action == 1:  # LONG
               if self.current_position == 0:
                   self.current_position = 1
                   self.entry_price = self._current_price()
                   trade_info = {"action": "open_long", "price": self.entry_price}

           elif action == 2:  # SHORT
               if self.current_position == 0:
                   self.current_position = -1
                   self.entry_price = self._current_price()
                   trade_info = {"action": "open_short", "price": self.entry_price}

           # Risk penalty (CC-14)
           risk_penalty = 0.001 * abs(self.current_position * self._current_price() * self.lot_size)
           reward -= risk_penalty

           return reward, trade_info

  Instruction 6 — ActorCritic Network Architecture
  ──────────────────────────────────────────────────────────────────────

  6.1  Network definition (network.py):

       class ActorCritic(nn.Module):
           def __init__(self, obs_dim: int = 23, action_dim: int = 3):
               super().__init__()
               self.shared = nn.Sequential(
                   nn.Linear(obs_dim, 128),
                   nn.ReLU(),
                   nn.Linear(128, 128),
                   nn.ReLU(),
               )
               self.actor = nn.Sequential(
                   nn.Linear(128, 64),
                   nn.ReLU(),
                   nn.Linear(64, action_dim),
               )
               self.critic = nn.Sequential(
                   nn.Linear(128, 64),
                   nn.ReLU(),
                   nn.Linear(64, 1),
               )
               # Orthogonal initialization (standard in PPO)
               self._init_weights()

           def _init_weights(self):
               for module in self.modules():
                   if isinstance(module, nn.Linear):
                       nn.init.orthogonal_(module.weight, gain=np.sqrt(2))
                       nn.init.constant_(module.bias, 0.0)
               # Last actor layer: small gain for near-uniform initial policy
               nn.init.orthogonal_(self.actor[-1].weight, gain=0.01)

           def forward(self, obs, mask=None):
               hidden = self.shared(obs)
               logits = self.actor(hidden)
               value = self.critic(hidden)
               if mask is not None:
                   logits[~mask] = -1e10  # Mask invalid actions (CC-07)
               return logits, value

  6.2  Architecture decisions:
       * Shared backbone (128->128) + separate actor/critic heads
       * 2-layer MLP is standard for financial RL (no spatial/temporal structure)
       * No LSTM/GRU — adds complexity, harder to export to ONNX,
         and 75-step episodes don't need long memory
       * Orthogonal init with gain=sqrt(2) for hidden layers,
         gain=0.01 for actor final layer (Zai & Brown Ch.4)
       * Total params: ~25k — small enough to avoid overfitting on
         ~500 trading days of data

       class MaskedCategorical:
           """Categorical distribution with action masking (CC-07)."""
           def __init__(self, logits: torch.Tensor, mask: torch.Tensor):
               self.logits = logits.clone()
               self.logits[~mask] = -1e10  # Set invalid action logits to -inf approx
               self.probs = F.softmax(self.logits, dim=-1)
               self.log_probs = F.log_softmax(self.logits, dim=-1)

           def sample(self) -> torch.Tensor:
               return torch.multinomial(self.probs, 1).squeeze(-1)

           def log_prob(self, action: torch.Tensor) -> torch.Tensor:
               return self.log_probs.gather(-1, action.unsqueeze(-1)).squeeze(-1)

           def entropy(self) -> torch.Tensor:
               return -(self.probs * self.log_probs).sum(-1)

  8.1  PPOAgent class (agent.py) — ~300 LOC following CleanRL pattern:

       class PPOAgent:
           def __init__(self, config: PPOTrainSettings, env_config: DRLSettings):
               self.config = config
               self.env_config = env_config
               self.device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
               self.network = ActorCritic(
                   obs_dim=env_config.observation_dim,
                   action_dim=env_config.action_dim,
               ).to(self.device)
               self.optimizer = torch.optim.Adam(
                   self.network.parameters(),
                   lr=config.learning_rate,
                   eps=1e-5,
               )
               self.normalizer = RunningMeanStd(shape=(env_config.observation_dim,))

           def train(self):
               """Main training loop — runs on Colab T4."""
               obs, info = self.envs.reset(seed=self.env_config.seed)  # CC-19
               for global_step in range(1, self.config.total_timesteps + 1):
                   # 1. Collect rollout
                   obs, actions, log_probs, rewards, dones, values, masks = \
                       self._collect_rollout(obs, info)

                   # 2. Compute advantages (GAE, CC-13)
                   advantages, returns = self._compute_gae(rewards, values, dones)

                   # 3. Normalize advantages
                   advantages = (advantages - advantages.mean()) / \
                                (advantages.std() + 1e-8)

                   # 4. PPO update
                   self._ppo_update(obs, actions, log_probs, advantages, returns, masks)

                   # 5. Entropy schedule (CC-25)
                   progress = global_step / self.config.total_timesteps
                   self.entropy_coef = self.config.entropy_coef_start + \
                       (self.config.entropy_coef_end - self.config.entropy_coef_start) * progress

                   # 6. Checkpoint (CC-15)
                   if global_step % self.config.checkpoint_every == 0:
                       self._checkpoint(global_step)

                   # 7. Validation (every 500k steps)
                   if global_step % 500_000 == 0:
                       self._validate()

  8.2  Rollout storage (rollout.py):

       class RolloutBuffer:
           def __init__(self, num_steps: int, num_envs: int, obs_dim: int):
               self.obs = np.zeros((num_steps, num_envs, obs_dim), dtype=np.float32)
               self.actions = np.zeros((num_steps, num_envs), dtype=np.int64)
               self.log_probs = np.zeros((num_steps, num_envs), dtype=np.float32)
               self.rewards = np.zeros((num_steps, num_envs), dtype=np.float32)
               self.dones = np.zeros((num_steps, num_envs), dtype=np.bool_)
               self.values = np.zeros((num_steps, num_envs), dtype=np.float32)
               self.masks = np.zeros((num_steps, num_envs, 3), dtype=np.bool_)
               self.ptr = 0

           def insert(self, **kwargs):
               for k, v in kwargs.items():
                   getattr(self, k)[self.ptr] = v
               self.ptr += 1

           def get(self):
               return {k: v[:self.ptr] for k, v in self.__dict__.items()
                       if isinstance(v, np.ndarray) and k != "ptr"}

  8.3  GAE computation (CC-13):

       def _compute_gae(self, rewards, values, dones):
           advantages = np.zeros_like(rewards)
           last_gae = 0
           for t in reversed(range(len(rewards))):
               if t == len(rewards) - 1:
                   next_value = 0  # Bootstrap from value function if not done
                   next_non_terminal = 1.0 - dones[t]
               else:
                   next_value = values[t + 1]
                   next_non_terminal = 1.0 - dones[t]
               delta = rewards[t] + self.config.gamma * next_value * next_non_terminal - values[t]
               advantages[t] = last_gae = delta + self.config.gamma * \
                   self.config.gae_lambda * next_non_terminal * last_gae
           returns = advantages + values
           return advantages, returns

  8.4  PPO update (clipped surrogate, CC-12, CC-26):

       def _ppo_update(self, obs, actions, old_log_probs, advantages, returns, masks):
           # Flatten batches
           b_obs = torch.tensor(obs.reshape(-1, *obs.shape[2:]), device=self.device)
           b_actions = torch.tensor(actions.flatten(), device=self.device)
           b_old_log_probs = torch.tensor(old_log_probs.flatten(), device=self.device)
           b_advantages = torch.tensor(advantages.flatten(), device=self.device)
           b_returns = torch.tensor(returns.flatten(), device=self.device)
           b_masks = torch.tensor(masks.reshape(-1, *masks.shape[2:]), device=self.device)

           for epoch in range(self.config.num_epochs):
               # Mini-batch updates
               for idx in np.random.permutation(len(b_obs))[::self.config.num_minibatches]:
                   mb_obs = b_obs[idx:idx+self.config.num_minibatches]
                   mb_actions = b_actions[idx:idx+self.config.num_minibatches]
                   mb_old_log_probs = b_old_log_probs[idx:idx+self.config.num_minibatches]
                   mb_advantages = b_advantages[idx:idx+self.config.num_minibatches]
                   mb_returns = b_returns[idx:idx+self.config.num_minibatches]
                   mb_masks = b_masks[idx:idx+self.config.num_minibatches]

                   # Forward pass
                   logits, value = self.network(mb_obs, mask=mb_masks)
                   dist = MaskedCategorical(logits, mb_masks)
                   log_prob = dist.log_prob(mb_actions)
                   entropy = dist.entropy().mean()

                   # Clipped surrogate (PPO)
                   ratio = (log_prob - mb_old_log_probs).exp()
                   surr1 = ratio * mb_advantages
                   surr2 = torch.clamp(ratio, 1 - self.config.clip_epsilon,
                                       1 + self.config.clip_epsilon) * mb_advantages
                   policy_loss = -torch.min(surr1, surr2).mean()

                   # Value loss
                   value_loss = 0.5 * ((value.squeeze() - mb_returns) ** 2).mean()

                   # Total loss (Sutton & Barto Ch.13)
                   loss = policy_loss - self.entropy_coef * entropy + \
                          self.config.value_loss_coef * value_loss

                   # Gradient step
                   self.optimizer.zero_grad()
                   loss.backward()
                   nn.utils.clip_grad_norm_(          # CC-12
                       self.network.parameters(),
                       self.config.max_grad_norm,
                   )
                   self.optimizer.step()

               # Early stopping on KL divergence (CC-26)
                   with torch.no_grad():
                       approx_kl = (ratio - 1) - (log_prob - mb_old_log_probs)
                       if approx_kl.mean() > self.config.target_kl:
                           break  # Early stop this epoch

  10.1  Inference class (inference.py):

       class DRLInference:
           def __init__(self, model_path: str, norm_stats_path: str, config: DRLSettings):
               self.session = onnxruntime.InferenceSession(
                   model_path,
                   providers=["CPUExecutionProvider"],  # CC-18
               )
               self.normalizer = InferenceNormalizer.from_json(norm_stats_path)  # CC-06
               self.config = config

           def predict(self, observation: np.ndarray, mask: np.ndarray) -> tuple[int, float]:
               # 1. Normalize observation (CC-06)
               norm_obs = self.normalizer.normalize(observation).astype(np.float32)

               # 2. Run inference
               inputs = {
                   "observation": norm_obs.reshape(1, -1),
                   "mask": mask.reshape(1, -1).astype(bool),
               }
               logits, value = self.session.run(None, inputs)

               # 3. Sample action from masked distribution
               logits = logits[0]
               logits[~mask] = -1e10
               probs = np.exp(logits) / np.exp(logits).sum()
               action = np.random.choice(3, p=probs)

               return int(action), float(value[0, 0])

  10.2  Inference contract:
       * Input: raw (unnormalized) observation + boolean mask
       * Output: action (int 0/1/2) + value estimate (float)
       * Latency: <50ms on Windows 11 CPU (CC-04)
       * No torch import in inference.py (CC-18)
       * Thread-safe for concurrent calls (ONNX Runtime session is thread-safe)

  11.1  WeightFSM (weight_fsm.py):

       States and transitions:
       DISABLED(0%) --OOS Sharpe>=0.5 for 5d--> REDUCED(5%)
       REDUCED(5%)  --OOS Sharpe>=0.8 for 20d--> ACTIVE(15%)
       ACTIVE(15%)  --OOS Sharpe>=1.0 for 30d--> ESCALATED_1(20%)
       ESCALATED_1(20%) --OOS Sharpe>=1.5 for 30d--> ESCALATED_2(25%)
       Any state    --OOS Sharpe<0.5 for 10d--> REDUCED(5%) (auto-reduce)

  11.2  Degradation detection (validation/degradation.py):

       class DegradationDetector:
           """Monitors OOS performance and triggers weight reduction."""
           def __init__(self, window: int = 10, threshold: float = 0.5):
               self.window = window
               self.threshold = threshold
               self.recent_sharpes: list[float] = []

           def update(self, daily_sharpe: float) -> str | None:
               self.recent_sharpes.append(daily_sharpe)
               if len(self.recent_sharpes) > self.window:
                   self.recent_sharpes = self.recent_sharpes[-self.window:]
               if len(self.recent_sharpes) >= self.window:
                   avg = sum(self.recent_sharpes) / len(self.recent_sharpes)
                   if avg < self.threshold:
                       return "DEGRADE"
               return None

  11.3  WeightFSM implementation:

       class WeightFSM:
           STATES = {
               "DISABLED": 0.0,
               "REDUCED": 0.05,
               "ACTIVE": 0.15,
               "ESCALATED_1": 0.20,
               "ESCALATED_2": 0.25,
           }

           def __init__(self, initial_state: str = "DISABLED"):
               self.state = initial_state
               self.consecutive_good_days = 0
               self.degradation_detector = DegradationDetector()

           @property
           def weight(self) -> float:
               return self.STATES[self.state]

           def transition(self, daily_sharpe: float) -> float:
               degrade = self.degradation_detector.update(daily_sharpe)

               if degrade == "DEGRADE":
                   self.state = "REDUCED"
                   self.consecutive_good_days = 0
               elif daily_sharpe >= self._threshold_for_escalation():
                   self.consecutive_good_days += 1
                   if self._should_escalate():
                       self._escalate()
               else:
                   self.consecutive_good_days = 0

               return self.weight

           def _threshold_for_escalation(self) -> float:
               thresholds = {
                   "DISABLED": 0.5,
                   "REDUCED": 0.8,
                   "ACTIVE": 1.0,
                   "ESCALATED_1": 1.5,
                   "ESCALATED_2": float("inf"),
               }
               return thresholds[self.state]

           def _should_escalate(self) -> bool:
               required_days = {
                   "DISABLED": 5,
                   "REDUCED": 20,
                   "ACTIVE": 30,
                   "ESCALATED_1": 30,
                   "ESCALATED_2": 0,
               }
               return self.consecutive_good_days >= required_days[self.state]

           def _escalate(self):
               order = ["DISABLED", "REDUCED", "ACTIVE", "ESCALATED_1", "ESCALATED_2"]
               idx = order.index(self.state)
               if idx < len(order) - 1:
                   self.state = order[idx + 1]
                   self.consecutive_good_days = 0

       class DRLSignalProvider:
           """Produces SignalPayload for OrchestrationEngine."""
           def __init__(self, inference: DRLInference, feature_builder, weight_fsm: WeightFSM):
               self.inference = inference
               self.feature_builder = feature_builder
               self.weight_fsm = weight_fsm

           def fetch(self, market_data: dict) -> SignalPayload:
               obs = self.feature_builder.build(market_data)
               mask = self.feature_builder.get_mask(market_data)
               action, value = self.inference.predict(obs, mask)

               # Convert action to signal
               direction_map = {0: "FLAT", 1: "BULLISH", 2: "BEARISH"}
               confidence = min(abs(value) / 2.0, 1.0)  # Value as proxy for confidence

               return SignalPayload(
                   source=SignalSource.DRL,
                   direction=direction_map[action],
                   confidence=confidence,
                   weight=self.weight_fsm.weight,
                   timestamp=datetime.now(IST),
                   metadata={"action": action, "value": value},
               )
