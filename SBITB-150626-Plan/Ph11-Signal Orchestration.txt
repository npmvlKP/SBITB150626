### 1.2 Validation Rules:
- Sum of all 5 WEIGHT_* fields must equal 1.0 (±0.001 tolerance for float arithmetic)
- Sum of all 5 TARGET_WEIGHT_* fields must equal 1.0 (±0.001 tolerance)
- `CONFIDENCE_THRESHOLD` must be >= 0.0 and <= 1.0
- `OPPOSITION_GATE_THRESHOLD` must be < `CONFIDENCE_THRESHOLD` (opposition gate is a softer check)
- `MIN_SOURCES_REQUIRED` must be >= 1 and <= 5
- Add a `@model_validator(mode="after")` to enforce cross-field validation:

```python
@model_validator(mode="after")
def validate_weights(self) -> "OrchestrationSettings":
    weight_sum = (
        self.WEIGHT_TECHNICAL + self.WEIGHT_VOLUME + self.WEIGHT_SENTIMENT
        + self.WEIGHT_MARKET_STRENGTH + self.WEIGHT_RAG
    )
    if abs(weight_sum - 1.0) > 0.001:
        raise ValueError(f"Signal weights must sum to 1.0, got {weight_sum:.4f}")
    target_sum = (
        self.TARGET_WEIGHT_TECHNICAL + self.TARGET_WEIGHT_VOLUME
        + self.TARGET_WEIGHT_SENTIMENT + self.TARGET_WEIGHT_MARKET_STRENGTH
        + self.TARGET_WEIGHT_RAG
    )
    if abs(target_sum - 1.0) > 0.001:
        raise ValueError(f"Target weights must sum to 1.0, got {target_sum:.4f}")
    if self.OPPOSITION_GATE_THRESHOLD >= self.CONFIDENCE_THRESHOLD:
        raise ValueError(
            f"OPPOSITION_GATE_THRESHOLD ({self.OPPOSITION_GATE_THRESHOLD}) must be < "
            f"CONFIDENCE_THRESHOLD ({self.CONFIDENCE_THRESHOLD})"
        )
    return self
```

### 3.1 Class ConfidenceCombiner

```python
class ConfidenceCombiner:
    """Combine multiple signal payloads into a single weighted score.

    Implements:
    - Weighted average with normalization for missing sources (Correction #9, #19)
    - Opposition gate: block if any source strongly opposes (Correction #11)
    - Data quality gate: require minimum sources (Correction #11)
    - Drop-one importance analysis (López de Prado Ch.9)

    Args:
        settings: OrchestrationSettings for weights and thresholds.
    """

    def __init__(self, settings: OrchestrationSettings):
        self._settings = settings
        self._source_weights: dict[SignalSource, float] = {
            SignalSource.TECHNICAL: settings.WEIGHT_TECHNICAL,
            SignalSource.VOLUME: settings.WEIGHT_VOLUME,
            SignalSource.SENTIMENT: settings.WEIGHT_SENTIMENT,
            SignalSource.MARKET_STRENGTH: settings.WEIGHT_MARKET_STRENGTH,
            SignalSource.RAG: settings.WEIGHT_RAG,
        }

    def combine(self, payloads: list[SignalPayload]) -> CombinedScore:
        """Combine signal payloads into a weighted score.

        Steps:
        1. Separate available (non-stub) vs missing (stub/absent) sources
        2. If available sources < MIN_SOURCES_REQUIRED: set combined_score = 0.0,
           all gates will FAIL_DATA_QUALITY
        3. Normalize weights: redistribute missing source weights proportionally
           among available sources so they sum to 1.0
           Formula: normalized_weight_i = weight_i / sum(available_weights)
        4. Compute weighted average:
           combined_score = sum(normalized_weight_i * score_i) for available sources
        5. Compute drop-one importance (López de Prado Ch.9):
           For each available source, recompute score with that source dropped.
           importance_i = |full_score - score_without_i|
        6. Return CombinedScore with all metadata

        CRITICAL (Correction #10): score range is [-1.0, +1.0].
        The threshold check must use abs(combined_score) > CONFIDENCE_THRESHOLD,
        NOT combined_score > CONFIDENCE_THRESHOLD.
        A bearish trade with combined_score = -0.75 should pass threshold gate.
        """

    def check_threshold_gate(self, combined: CombinedScore) -> DecisionGateResult:
        """Gate 1: Is |combined_score| > CONFIDENCE_THRESHOLD?

        (Correction #10: uses abs() for bearish signals)
        Returns PASS or FAIL_THRESHOLD.
        """
        if abs(combined.combined_score) > self._settings.CONFIDENCE_THRESHOLD:
            return DecisionGateResult.PASS
        return DecisionGateResult.FAIL_THRESHOLD

    def check_opposition_gate(self, combined: CombinedScore) -> DecisionGateResult:
        """Gate 2: Does any source strongly oppose the combined direction?

        (Correction #11): A source opposes if:
        - sign(source_score) != sign(combined_score), AND
        - |source_weight * source_score| > OPPOSITION_GATE_THRESHOLD

        This prevents trading when a single source has a strong contrarian view.
        Returns PASS or FAIL_OPPOSITION.
        """

    def check_data_quality_gate(self, combined: CombinedScore) -> DecisionGateResult:
        """Gate 3: Are enough sources available?

        (Correction #11): n_sources_available >= MIN_SOURCES_REQUIRED
        Returns PASS or FAIL_DATA_QUALITY.
        """

    def _normalize_weights(self, available_sources: set[SignalSource]) -> dict[SignalSource, float]:
        """Redistribute weights from missing sources to available sources.

        Formula: normalized_w_i = raw_w_i / sum(raw_w_j for j in available)

        This ensures weights always sum to 1.0 regardless of missing sources.
        """

    def _compute_drop_one_importance(self, payloads: list[SignalPayload], full_score: float) -> dict[SignalSource, float]:
        """Compute feature importance via drop-one analysis.

        For each available source i:
        1. Remove source i from payloads
        2. Recompute combined score without source i
        3. importance_i = |full_score - score_without_i|

        Per López de Prado Ch.9: this measures how much each source
        contributes to the final decision. High importance = high influence.
        """
```

### 3.2 Key Implementation Rules:
- `combine()` must NEVER raise an exception — if all sources are missing, return `CombinedScore(combined_score=0.0, ...)` with `n_sources_available=0`
- Weight normalization: if only TECHNICAL and VOLUME are available (weights 0.20 each), normalized = 0.50 each (0.20/0.40)
- Opposition gate: only check sources whose `is_stub=False` — stub sources have no real opinion
- Drop-one importance is computed ONLY for non-stub sources
- All intermediate calculations use float; final results stored in frozen dataclass

class SignalFetcher:
    """Fetch all signal sources in parallel with circuit breaker protection.

    Uses asyncio.gather() for parallel fetching (Correction #18).

        market_strength_engine: Phase 4 MarketStrengthEngine (or None if not implemented).

        data_provider: MarketDataProvider | None = None,
        analysis_engine: object | None = None,
        market_strength_engine: object | None = None,

        self._data_provider = data_provider
        self._analysis_engine = analysis_engine
        self._market_strength_engine = market_strength_engine

        Steps:
        1. For each SignalSource, create an asyncio task for _fetch_source()
        2. Run all tasks via asyncio.gather(*tasks, return_exceptions=True)
        3. For any task that raised an exception:
           - Record failure in circuit breaker
           - Return stub payload for that source
        4. For any task where circuit breaker was OPEN:
           - Skip the fetch entirely, return stub immediately
        5. Return list of 5 SignalPayloads (real or stub)

        Timeout: each individual fetch has a 5-second timeout.

    async def _fetch_source(self, source: SignalSource, symbol: str) -> SignalPayload:

        - TECHNICAL: call analysis_engine.generate_technical_report(symbol)

        - MARKET_STRENGTH: call market_strength_engine.evaluate(symbol)

        If the engine/pipeline is None (phase not implemented): return stub with score=0.0, confidence=0.0
        If the engine/pipeline raises an exception: re-raise (handled by fetch_all)

        RAG conversion (Correction #14):
        - response = rag_pipeline.query(f"Market outlook for {symbol}")
        - Parse structured response for BULLISH/NEUTRAL/BEARISH keyword
        - score = RAG_BULLISH_SCORE / RAG_NEUTRAL_SCORE / RAG_BEARISH_SCORE
        - confidence = 0.5 (RAG is inherently lower confidence than quantitative)
        """

    def _make_stub(self, source: SignalSource, symbol: str, reason: str) -> SignalPayload:
        """Create a stub payload for an unavailable source.

        Stub: score=0.0, confidence=0.0, is_stub=True
        Reason recorded in metadata["stub_reason"].
        """

    def get_circuit_breaker_status(self, source: SignalSource) -> CircuitBreakerStatus:
        """Get current circuit breaker status for a source."""

    def reset_circuit_breaker(self, source: SignalSource) -> None:
        """Manually reset a circuit breaker to CLOSED state.

### 4.2 Class _CircuitBreaker (internal)

```python
class _CircuitBreaker:
    """Circuit breaker per signal source (Kleppmann Ch.7).

    States:
    - CLOSED: Normal. Track consecutive failures.
      On CIRCUIT_BREAKER_FAILURE_THRESHOLD consecutive failures -> transition to OPEN.
    - OPEN: Reject all requests immediately.
      After CIRCUIT_BREAKER_OPEN_DURATION_SECONDS -> transition to HALF_OPEN.
    - HALF_OPEN: Allow CIRCUIT_BREAKER_HALF_OPEN_MAX_CALLS requests.
      If they succeed -> transition to CLOSED.
      If any fails -> transition back to OPEN.

    Thread safety: use asyncio.Lock for state transitions.

    Args:
        source: The signal source this breaker protects.
        settings: OrchestrationSettings for thresholds.
    """

    def __init__(self, source: SignalSource, settings: OrchestrationSettings):
        self._source = source
        self._settings = settings
        self._state = CircuitBreakerState.CLOSED
        self._consecutive_failures = 0
        self._last_failure_time: float | None = None
        self._last_failure_reason: str | None = None
        self._open_until: float | None = None
        self._half_open_calls: int = 0
        self._total_failures = 0
        self._total_successes = 0
        self._lock = asyncio.Lock()

    async def before_call(self) -> bool:
        """Check if a call is allowed.

        Returns:
            True if the call should proceed, False if circuit is OPEN.
        In HALF_OPEN, increments call counter and allows limited calls.
        """

    async def record_success(self) -> None:
        """Record a successful call.

        In CLOSED: reset consecutive_failures to 0.
        In HALF_OPEN: if enough successes, transition to CLOSED.
        """

    async def record_failure(self, reason: str) -> None:
        """Record a failed call.

        In CLOSED: increment consecutive_failures.
          If >= threshold, transition to OPEN.
        In HALF_OPEN: immediately transition back to OPEN.
        """

    def get_status(self) -> CircuitBreakerStatus:
        """Return current status as CircuitBreakerStatus dataclass."""
```

### 4.3 Key Implementation Rules:
- `fetch_all()` uses `asyncio.gather(*tasks, return_exceptions=True)` — never let one source's failure crash others
- Each source fetch has a 5-second timeout: `asyncio.wait_for(_fetch_source(), timeout=5.0)`
- If `analysis_engine` is None, TECHNICAL and VOLUME return stubs (Phase 3 not implemented)
- If `market_strength_engine` is None, MARKET_STRENGTH returns stub (Phase 4 not implemented)
- If `sentiment_pipeline` is None, SENTIMENT returns stub (Phase 8 not implemented)
- If `rag_pipeline` is None, RAG returns stub (Phase 9 not implemented)
- All stubs have `is_stub=True`, `score=0.0`, `confidence=0.0`
- Circuit breaker state transitions are logged via structlog
- NEVER retry a fetch within the same eval cycle — one attempt per source per cycle

### 5.2 Class TrailingStopMonitor

```python
class TrailingStopMonitor:
    """Ratchet trailing stop monitor for active positions.

    Implements ratchet mechanism (Correction #13):
    - Stop only moves in favorable direction (up for long, down for short)
    - Stop NEVER retreats — once advanced, stays advanced
    - Ratchet advances only when price moves >= min_move_pct from previous best
    - When stop is hit, generates an EXIT TradeDecision

    Per Kaufman Ch.10: trailing stops must adapt to volatility regime.
    The stop_distance_pct is set at entry by Phase 5's AdaptiveTrailingStop
    (which adjusts by VIX + market_strength + volume_spike).
    Phase 11's monitor only executes the ratchet — it does NOT recompute distance.

    Args:
        settings: OrchestrationSettings for TRAILING_STOP_MIN_MOVE_PCT.
    """

    def __init__(self, settings: OrchestrationSettings):
        self._settings = settings
        self._positions: dict[str, TrailingStopState] = {}  # symbol -> state

    def register_position(
        self,
        symbol: str,
        direction: int,
        entry_price: float,
        stop_distance_pct: float,
        position_id: str | None = None,
    ) -> TrailingStopState:
        """Register a new position for trailing stop monitoring.

        Steps:
        1. Create TrailingStopState with initial stop price
        2. For long: initial_stop = entry_price * (1 - stop_distance_pct)
        3. For short: initial_stop = entry_price * (1 + stop_distance_pct)
        4. Store in self._positions[symbol]
        5. Return the state

        Raises ValueError if symbol already has an active position.
        """

    def update_tick(self, symbol: str, current_price: float, timestamp: float) -> TradeDecision | None:
        """Update trailing stop on a new tick.

        Steps:
        1. If symbol not in _positions, return None
        2. Get TrailingStopState for symbol
        3. For long (direction=+1):
           a. If current_price > best_price:
              - Check if (current_price - best_price) / best_price >= min_move_pct
              - If yes: ratchet stop up to current_price * (1 - stop_distance_pct)
              - Update best_price = current_price
              - Increment n_ratchet_advances
           b. If current_price <= current_stop_price:
              - STOP HIT: generate EXIT TradeDecision
        4. For short (direction=-1):
           a. If current_price < best_price:
              - Check if (best_price - current_price) / best_price >= min_move_pct
              - If yes: ratchet stop down to current_price * (1 + stop_distance_pct)
              - Update best_price = current_price
              - Increment n_ratchet_advances
           b. If current_price >= current_stop_price:
              - STOP HIT: generate EXIT TradeDecision

        Ratchet rule (Correction #13): stop NEVER retreats.
        If computed new stop is worse than current stop, keep current stop.

        Returns: TradeDecision with action="EXIT" if stop hit, None otherwise.
        """

    def deregister_position(self, symbol: str) -> TrailingStopState | None:
        """Remove a position from monitoring after exit."""

    def get_state(self, symbol: str) -> TrailingStopState | None:
        """Get current trailing stop state for a symbol."""

    def get_all_states(self) -> dict[str, TrailingStopState]:
        """Get all active trailing stop states."""
```

### 5.3 Key Implementation Rules:
- Ratchet ONLY advances — never retreats. If `new_stop < current_stop` for long (or `new_stop > current_stop` for short), keep `current_stop`
- `min_move_pct` prevents micro-ratcheting on noise (0.3% default = ~30 points on NIFTY at 24000)
- Stop hit generates a `TradeDecision` with `action="EXIT"` — it does NOT place an order
- All ratchet advances and stop hits are logged via structlog
- If `current_price` is 0 or negative, skip update (data quality protection)
- Thread safety: this monitor is used from the orchestrator's eval loop — single-threaded access per symbol (no lock needed)

        self,
        symbol: str,
        settings: OrchestrationSettings,
        signal_fetcher: SignalFetcher,
        confidence_combiner: ConfidenceCombiner,
        trailing_stop_monitor: TrailingStopMonitor,
        risk_manager: RiskManager | None = None,
        audit_logger: AuditLogger | None = None,
        strategy: StrategyInterface | None = None,
        kill_switch: KillSwitch | None = None,
    ):
        self._symbol = symbol
        self._settings = settings
        self._fetcher = signal_fetcher
        self._combiner = confidence_combiner
        self._trailing_stop = trailing_stop_monitor
        self._risk_manager = risk_manager
        self._audit_logger = audit_logger
        self._strategy = strategy
        self._kill_switch = kill_switch

        self._eval_count: int = 0
        self._trade_count: int = 0
        self._hold_count: int = 0
        self._gate_fail_count: dict[DecisionGateResult, int] = {}
        self._is_running = False

    async def evaluate(self, current_time: float) -> TradeDecision | None:
        """Run one evaluation cycle for this symbol.

        Steps:
        1. Check kill switch: if KILL or PAUSE, return HOLD decision
        2. Throttle eval: skip if (current_time - last_eval_time) < SIGNAL_EVAL_INTERVAL_SECONDS
        3. Fetch all signals via signal_fetcher.fetch_all(symbol)
        4. Combine signals via confidence_combiner.combine(payloads)
        5. Run Gate 1 (threshold): check_threshold_gate()
        6. Run Gate 2 (opposition): check_opposition_gate()
        7. Run Gate 3 (data quality): check_data_quality_gate()
        8. If ANY gate fails:
           - Log gate failure reason
           - Return TradeDecision with action="HOLD" and gate_result=FAIL_*
           - Increment gate_fail_count for failed gate
        9. If ALL gates pass:
           a. Determine direction: combined_score > 0 -> BUY_LONG, < 0 -> SELL_SHORT
           b. If strategy is available: get strike, lots, stop_distance from strategy
           c. If strategy is None: use defaults (ATM strike, 1 lot, 0.03 stop distance)
           d. Run risk_manager.pre_trade_check() if available
           e. If risk check FAIL/THROTTLED/KILLED: return HOLD with risk result
           f. Register trailing stop for new position
           g. Create TradeDecision with all fields populated
           h. Log to audit trail
           i. Return TradeDecision
        10. Update eval_count, last_eval_time
        """

    async def check_exit_signals(self, current_price: float, current_time: float) -> TradeDecision | None:
        """Check if any active position should exit.

        Exit triggers (Plan.txt line 695):
        1. Trailing stop hit (checked by TrailingStopMonitor.update_tick)
        2. Regime change (from MarketStrengthEngine — if available)
        3. Time stop (close to expiry — Phase 5 logic)
        4. Target achieved (2:1 R:R for buying, 50% premium decay for selling)

        Returns TradeDecision with action="EXIT" if any trigger fires, None otherwise.
        """

    def get_stats(self) -> dict:
        """Get orchestrator statistics for monitoring.

        Returns dict with:
        - symbol, eval_count, trade_count, hold_count
        - gate_fail_count by gate type
        - circuit_breaker_status per source
        - trailing_stop_states
        - is_running
        """
```

### 6.2 Key Implementation Rules:
- `evaluate()` is called from SessionManager's eval loop — NOT directly from tick handler

- Throttle: skip if interval hasn't elapsed — prevents over-evaluation on rapid ticks
- If `risk_manager` is None, skip risk check (Phase 10 may not be fully implemented)
- If `strategy` is None, use default strike/lot values — orchestrator must work without Phase 5
- If `audit_logger` is None, skip audit logging (graceful degradation)

- `gate_fail_count` tracks gate failures for monitoring dashboards
- The orchestrator NEVER calls broker.place_order() — it only creates TradeDecision DTOs (Correction #15)
- Per-symbol isolation: one orchestrator instance per symbol, no shared mutable state

class SessionManager:

       every SIGNAL_EVAL_INTERVAL_SECONDS.
    3. POST_CLOSE (15:30-15:40 IST): P&L logging, position summary,
       audit trail finalization, cleanup.
    4. CLOSED: Bot is idle. No operations.

    Args:
        settings: OrchestrationSettings.
        orchestrators: Dict of symbol -> SignalOrchestrator.
        compliance_settings: ComplianceSettings for session time checks.
        audit_logger: AuditLogger for session events.
        kill_switch: KillSwitch for emergency halt.
    """

    def __init__(
        self,
        settings: OrchestrationSettings,
        orchestrators: dict[str, SignalOrchestrator],
        compliance_settings: ComplianceSettings,
        audit_logger: AuditLogger | None = None,
        kill_switch: KillSwitch | None = None,
    ):
        self._settings = settings
        self._orchestrators = orchestrators
        self._compliance = compliance_settings
        self._audit_logger = audit_logger
        self._kill_switch = kill_switch
        self._session_start_time: float | None = None
        self._session_end_time: float | None = None
        self._daily_pnl: Decimal = Decimal("0")
        self._daily_trades: int = 0
        self._is_running = False

    async def run_session(self) -> None:
        """Run the full trading session from PRE_OPEN through POST_CLOSE.

        This is the main entry point called by the bot's main loop.

        Steps:
        1. Wait for PRE_OPEN time (9:00 IST minus PRE_OPEN_WARMUP_SECONDS)
        2. Run warmup phase:
           a. Fetch initial market data for all symbols
           b. Pre-compute technical indicators (if Phase 3 available)
           c. Load RAG macro context (if Phase 9 available)
           d. Log "warmup_complete" to audit trail
        3. Wait for REGULAR session (9:15 IST)
        4. Run trading loop:
           while session == REGULAR:
             a. For each orchestrator: await evaluate(current_time)
             b. For each orchestrator: await check_exit_signals(current_price, current_time)
             c. Sleep for SIGNAL_EVAL_INTERVAL_SECONDS (or remaining interval)
             d. Check kill switch status
        5. On POST_CLOSE (15:30 IST):
           a. Log daily P&L summary
           b. Log daily trade count and gate failure stats
           c. Finalize audit trail
           d. Clear trailing stop states
        6. Return (session complete)
        """

    async def _warmup_phase(self) -> None:
        """PRE_OPEN warmup: fetch initial data, establish connections.

        Per Correction #17: warmup starts PRE_OPEN_WARMUP_SECONDS before 9:15.
        Fetches initial data for all enabled symbols.
        Does NOT make any trade decisions.
        """

    async def _trading_loop(self) -> None:
        """Main trading loop during REGULAR session.

        Per Correction #12: eval every SIGNAL_EVAL_INTERVAL_SECONDS (default 5 min).
        Per Correction #16: each symbol has its own orchestrator.
        Per Correction #15: orchestrator creates TradeDecision DTOs, not broker calls.
        """

    async def _post_close_phase(self) -> None:
        """POST_CLOSE: P&L logging, audit finalization, cleanup.

        Per Correction #17: runs for POST_CLOSE_CLEANUP_SECONDS after 15:30.
        Logs:
        - Daily realized + unrealized P&L
        - Total trades, holds, gate failures
        - Circuit breaker statistics
        - Trailing stop ratchet statistics
        """

    def get_session_stats(self) -> dict:
        """Get current session statistics.

        Returns dict with:
        - session_duration_seconds
        - daily_pnl, daily_trades
        - per-symbol stats (from each orchestrator.get_stats())
        - kill_switch_status
        """
```

### 7.2 Key Implementation Rules:
- `run_session()` is a long-running coroutine — called once from the bot's main entry point
- Warmup does NOT trade — only prepares data and connections
- Trading loop checks `get_trading_session()` on each iteration to detect session transitions
- If kill switch is activated during trading loop, immediately exit loop to POST_CLOSE

- All phase transitions are logged via structlog and audit trail

## Appendix B — Circuit Breaker State Machine

```
CLOSED ──[failure_count >= threshold]──> OPEN
   ^                                        |
   |                                        |
   |                                        v
   └──[success in half-open]── HALF_OPEN <──[open_duration elapsed]
                                        |
                                        └──[failure in half-open]──> OPEN
```

Per Kleppmann Ch.7: Circuit breaker prevents cascading failures by failing fast
when a downstream service (signal source) is unhealthy.

Weight optimization method (Phase 6):
1. Walk-forward backtesting with 6-month rolling windows
2. Out-of-sample validation on each window
3. Drop-one importance analysis to identify noisy sources
4. Weight adjustment based on Sharpe contribution per source
5. Final weights must pass stability test: performance doesn't degrade ±10% from optimal

Per López de Prado Ch.9: avoid overfitting by using at least 3 independent out-of-sample periods.
