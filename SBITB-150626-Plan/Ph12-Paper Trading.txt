### Natenberg "Option Volatility and Pricing"

| # | Chapter | Extraction | Phase 12 Application |
|---|---------|-----------|---------------------|
| 1 | Ch.19 | "Theoretical price assumes perfect fill — real fills depend on bid-ask spread, market depth, and order urgency" | Fill simulator uses bid-ask midpoint + configurable slippage. Never assume fill at theoretical price. Illiquid strikes get higher slippage |
| 2 | Ch.19 | "Options slippage is asymmetric: buying pressure widens the ask, selling pressure widens the bid" | Directional slippage model: buying options = fill closer to ask (slippage = 60% of spread), selling options = fill closer to bid (slippage = 40% of spread). Reflects market maker behavior |
| 3 | Ch.19 | "Near-expiry options have wider effective spreads due to gamma risk — market makers demand more premium for providing liquidity" | Near-expiry slippage multiplier: x2.0 for options within 3 DTE, x1.5 for 4-7 DTE. Applied on top of base slippage |

### 3.2 Class SlippageModel

```python
class SlippageModel:
    """Multi-factor slippage model for options fill simulation.

    Slippage = base_slippage x session_multiplier x near_expiry_multiplier x illiquidity_multiplier

    Reference: Natenberg Ch.19 (execution vs theoretical pricing)
    Reference: Kaufman Ch.12 (realistic slippage in trading systems)
    """

    def __init__(self, settings: PaperTradingSettings):
        self._settings = settings

    def compute_slippage(
        self,
        symbol: str,
        option_type: str,
        signal_price: float,
        bid: float,
        ask: float,
        dte: int,
        timestamp: float,
        is_buy: bool,
    ) -> SlippageResult:
        """Compute total slippage for a simulated fill.

        Steps:
        1. Determine base slippage: NIFTY = BASE_SLIPPAGE_NIFTY_PCT, BANKNIFTY = BASE_SLIPPAGE_BANKNIFTY_PCT
        2. Compute session time multiplier:
           - First 15 min (9:15-9:30 IST): SESSION_OPEN_CLOSE_MULTIPLIER
           - Last 15 min (15:15-15:30 IST): SESSION_OPEN_CLOSE_MULTIPLIER
           - Core hours (9:30-15:15 IST): 1.0
        3. Compute near-expiry multiplier:
           - dte <= 3: NEAR_EXPIRY_3DTE_MULTIPLIER
           - dte 4-7: NEAR_EXPIRY_7DTE_MULTIPLIER
           - dte > 7: 1.0
        4. Compute illiquidity multiplier:
           - spread_pct = (ask - bid) / midpoint > ILLIQUID_SPREAD_THRESHOLD_PCT: ILLIQUID_STRIKE_MULTIPLIER
           - Otherwise: 1.0
        5. Total slippage = base x session_mult x near_expiry_mult x illiquidity_mult x signal_price
        6. Apply directional adjustment (Natenberg Ch.19):
           - Buy: fill = midpoint + (spread x BUY_SLIPPAGE_SPREAD_FRACTION) + total_slippage
           - Sell: fill = midpoint - (spread x SELL_SLIPPAGE_SPREAD_FRACTION) - total_slippage
        7. Return SlippageResult with all components for audit

        CRITICAL: slippage is ALWAYS unfavorable to the trader.
        """
```

### 3.3 Key Implementation Rules:
- Slippage is ALWAYS unfavorable — never reduces the trader's cost
- `compute_slippage()` must NEVER raise an exception — if inputs are invalid (price <= 0, bid > ask), return default slippage of 0.10% of signal_price and log a warning
- All multipliers are multiplicative (not additive): total = base x session x expiry x illiquidity
- If bid=0 or ask=0 (no market data), use conservative estimate: midpoint = signal_price, spread = 2x base slippage
- Session time is converted from UTC epoch to IST (UTC+5:30) for session category determination
- All slippage components are returned in SlippageResult for full audit trail

```python
        ask: float,
        dte: int,
    ) -> PaperFill:

        Exit-specific adjustments:
        - Kill switch exits: assume market order = worst case slippage x1.5
        - Trailing stop exits: fill at stop price (or slightly worse)
        """
```

### 8.2 Class CrashRecoveryManager

```python
class CrashRecoveryManager:
    """Recover paper trading state after crash or restart.

    Per Kaufman Ch.12: trading systems must handle crashes gracefully.
    Per Correction #19: recovery target < 60 seconds from process restart.

    Recovery steps:
    1. Load last known state from TimescaleDB
    2. Verify state consistency
    3. Skip signals generated during downtime (NEVER backfill)
    4. Resume from current market state
    5. Log crash event with stack trace if available
    """

    def __init__(
        self,
        settings: PaperTradingSettings,
        db_store: object | None = None,
        order_manager: PaperOrderManager | None = None,
    ):
        self._settings = settings
        self._db_store = db_store
        self._order_manager = order_manager

    async def recover_state(self) -> RecoveryResult:
        """Recover paper trading state from database.

        If no state found (first run): return empty state with initial capital.
        If state is inconsistent: log CRITICAL, use conservative values.
        """

    def save_state(self, order_manager: PaperOrderManager) -> None:
        """Persist current state to TimescaleDB. Called every eval cycle."""
```

### 8.3 Key Implementation Rules:
- State is persisted every eval cycle (5 min default) — max data loss = 1 eval cycle
- Recovery NEVER backfills missed signals — this would introduce look-ahead bias
- Heartbeat table: insert row every HEARTBEAT_INTERVAL_SECONDS (30s) — used for downtime calculation
- On inconsistent state: use CONSERVATIVE values (lower capital, wider stops)
- Recovery target: < 60 seconds from process restart to first signal evaluation
- If recovery fails completely: start fresh with initial capital, log CRITICAL, and alert operator

```python
    2. Mid-Point review at end of Week 3 (comprehensive)

        2. Compute weekly/cumulative Sharpe from daily P&L returns
        3. Compute weekly/cumulative max drawdown
        4. Compute uptime from heartbeat data

        6. Compute average paper vs backtest divergence
        7. Compute DSR on cumulative data (Lopez de Prado Ch.1)

    def _compute_dsr(self, observed_sharpe: float, n_trials: int, skew: float, kurtosis: float, t: int) -> float:
        """DSR = Prob(Sharpe* > observed | N trials, skew, kurtosis, T). Same as Phase 6."""

    def _evaluate_early_stop(self) -> bool:
        """If last 2 weekly reviews both have verdict=NO_GO, return True (Davey Ch.7, Correction #10)."""

    def _determine_verdict(self, metrics: dict) -> GoNoGoVerdict:
```

```python
class SixWeekReportGenerator:
    """Generate the final 6-week paper trading report with go/no-go decision.

    This is the deliverable for Phase 12. The report contains:
    1. All 6 weekly reviews
    2. Cumulative metrics across all 6 weeks
    3. Go/No-Go decision with documented reasons

        self,

        order_manager: PaperOrderManager,

        self._order_manager = order_manager
        self._daily_results = daily_checklist_results
        self._settings = settings

    def generate_report(self) -> SixWeekReport:
        """Generate the final 6-week paper trading report.

        Steps:
        1. Aggregate all weekly reviews
        2. Compute cumulative metrics (Sharpe, DD, DSR, uptime, etc.)
        3. Determine final verdict with documented reasons
        4. Document known simulator biases (Lopez de Prado Ch.11)
        5. Write recommendation:
           - GO -> proceed to Phase 13 (live deployment with minimal capital)
           - CAUTION -> extend paper trading by 2-4 weeks OR reduce position size
           - NO_GO -> return to Phase 6 for strategy revision
        6. Persist report to TimescaleDB
        7. Write report to file: reports/paper_trading/six_week_report_YYYYMMDD.json
        8. Return SixWeekReport
        """

    def _document_simulator_biases(self) -> list[str]:
        """Document known simulator biases per Lopez de Prado Ch.11.

        Returns list of 6 known biases:
        1. Simulator bias: overconfidence from risk-free environment
        2. Liquidity illusion: fills assume available counterparties
        3. Slippage estimation: may underestimate real market impact
        4. No emotional factor: real trading induces stress-based errors
        5. Market impact: large orders don't move simulated prices
        6. Timing assumption: fills at signal time, not order book time
        """

    def _determine_final_verdict(self, metrics: dict) -> tuple[GoNoGoVerdict, list[str]]:
        """Determine final go/no-go verdict with documented reasons.

        Go conditions (ALL must be met):
        - total_trading_sessions >= MIN_TRADING_SESSIONS
        - total_completed_trades >= MIN_COMPLETED_TRADES
        - final_sharpe >= GO_SHARPE (0.8)
        - final_max_dd < GO_MAX_DD_PCT (25%)
        - final_dsr >= DSR_MIN_THRESHOLD (0.95)
        - final_uptime >= GO_UPTIME_PCT (99.5%)
        - total_unexplained_breaches == 0
        - kill_switch_test_pass_rate == 100%

        No-Go conditions (ANY triggers NO_GO):
        - final_sharpe < CAUTION_SHARPE (0.3)
        - final_max_dd > NOGO_MAX_DD_PCT (35%)
        - final_uptime < NOGO_UPTIME_PCT (98%)
        - total_unexplained_breaches > 0
        - kill_switch tests failing
        - Any critical unhandled errors
        """

    def write_report_to_file(self, report: SixWeekReport) -> str:
        """Write report to JSON file in reports/paper_trading/ directory."""
```
